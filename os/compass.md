# DecisionOS Compass

Version: 1.0

---

# Purpose

The Compass is the sensing system of DecisionOS.

Its purpose is to continuously determine whether the operating system is moving toward its intended destination.

The Compass does not:

- make decisions
- change priorities
- suggest improvements

It observes reality.

It measures reality.

It reports reality.

Nothing more.

---

# Responsibilities

The Compass is responsible for:

• Measuring progress

• Recording evidence

• Detecting drift

• Monitoring capacity

• Reporting current state

The Compass never attempts to solve problems.

It only describes them.

---

# Guiding Principle

> You cannot improve what you cannot accurately observe.

The Compass exists to answer one question:

"Where am I?"

---

# What Gets Measured

DecisionOS intentionally measures very little.

Every metric must improve decision quality.

If it does not,

remove it.

# Category 2

## Artifact Production

Artifacts represent reality.

Examples

Software shipped

Resume improved

Interview completed

Workout completed

Blog published

Bug fixed

Revenue earned

Investment made

Every artifact should be concrete.

Research is not an artifact.

Planning is not an artifact.

Thinking is not an artifact.

---

# Category 3

## Capacity

Capacity determines future execution.

DecisionOS tracks:

Sleep

Nutrition

Exercise

Recovery

Mental Energy

These are leading indicators.

Protect them.

---

# Category 4

## Attention Allocation

Question

Where did today's attention actually go?

Categories

Campaign

Oracle

Maintenance

Entertainment

Distraction

Admin

Learning

Planning

Execution

DecisionOS compares:

Planned Attention

vs

Actual Attention

This reveals drift.

# Drift Detection

The Compass identifies drift.

Examples

Mission repeatedly incomplete

Health declining

Research increasing

Artifacts decreasing

Sleep deteriorating

Entertainment replacing execution

Campaign neglected

Attention fragmentation

Drift is information.

Not failure.

# Reality Principle

The Compass never interprets.

Examples

Good

Mission incomplete.

Sleep averaged 6.3 hours.

Three artifacts created.

Research exceeded implementation.

Bad

"You were lazy."

"You weren't disciplined."

Those are interpretations.

Not observations.

---

Sleep

Recovery

Movement

Nutrition

Stress

The Compass does not prescribe solutions.

Only reports condition.

---

# Confidence

DecisionOS recognizes uncertainty.

Whenever information is incomplete,

say

Unknown

Never invent certainty.

---

# Dashboard

The dashboard should be generated directly from Compass data.

Dashboard reflects.

Dashboard never analyzes.

Analysis belongs elsewhere.

---

# Success

The Compass succeeds when it accurately represents reality.

Not when reality is good.

Accuracy is the objective.

---

# Final Principle

The Compass exists so that the operating system always knows where it is.

Only after reality is accurately observed should the operating system attempt to change course.
